Product Requirements Document: Enterprise AEO Platform
The Enterprise Answer Engine Optimization Platform is designed to act as the Google Search Console for the AI era. Companies currently spend heavily on traditional search optimization but lack visibility into how AI models perceive their products. This platform solves this by continuously evaluating website content and autonomously generating developer-ready code fixes to improve AI visibility.

This product strictly adheres to the Track B Enterprise Agent Engineering architecture. Development begins locally using the Gemini CLI and the Agent Development Kit to scaffold the agent logic. These agents are then registered into Nasiko, which serves as the central control plane and orchestration layer. Nasiko securely manages policy access and routes the execution directly to Google Vertex AI. The entire Nasiko-managed pipeline, along with the React-based user interface, is packaged and deployed to Google Cloud Run, fulfilling the mandatory live deployment criteria.

The core system relies on specialized agents communicating through the Nasiko control plane. The Knowledge Ingestion Agent first scrapes and structures the target company's website and documentation. Next, the User Intent Agent analyzes this data to generate a matrix of realistic user questions. The AI Testing and Evaluation Agent then uses Vertex AI to simulate answering these questions, scoring the outputs on confidence, hallucination risk, and missing information. A Competitor Intelligence Agent performs a similar analysis on rival websites to provide comparative benchmarks. Finally, the Content Gap Agent identifies the root causes of low scores, such as missing JSON-LD schemas or unstructured API documentation.

The standout feature bridging AI analysis with engineering output is the custom CLI tool built on top of the Gemini CLI workflow. Developers interact with the swarm directly from their terminal using specific scan commands to trigger the Nasiko-orchestrated evaluation process. When issues are flagged, developers run a fix command. The Remediation Architect Agent then analyzes the local codebase, identifies exact files like markdown documentation or React components, and generates the necessary code changes directly into the developer's local environment.

The web interface, hosted on Google Cloud Run, provides high-level observability for marketing and engineering leadership. It displays the overall AEO score tracking AI visibility over time, alongside metrics for question success rate and documentation trust. The dashboard also features a competitor benchmark for side-by-side comparisons and a remediation queue showing pending website fixes generated by the swarm awaiting human approval.

You are tasked with building this multi-agent application using Next.js for the frontend, Python FastAPI for the backend, and Nasiko for agent orchestration. Begin by setting up the Python environment and using the Agent Development Kit to scaffold the Ingestion, Evaluation, and Remediation agents. Register these agents into the Nasiko registry, configuring it to use Google Vertex AI as the LLM runtime. Next, build the Next.js dashboard using Tailwind CSS to display the AEO metrics fetched from the FastAPI backend. Finally, create a comprehensive Dockerfile that packages both the frontend and the Nasiko-orchestrated backend, ensuring all Vertex AI and Nasiko credentials are exposed as environment variables for deployment to Google Cloud Run.
